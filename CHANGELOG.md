# Changelog

이 문서는 `cms-orbit/announcement`의 릴리스 노트를 기록합니다.

## 4.0.0-beta4 - 2026-07-04

### 개선

- 이제 더 이상 공지 공개 화면에서 제목, 빈 상태, 조회 수, 목록 복귀 문구가 하드코딩된 한국어에 묶여 있지 않고, 한글팩 키를 통해 일관되게 노출되도록 정리했습니다.
- `cms-orbit/core` `4.0.0-beta4` 기준으로 패키지 버전선을 맞춰 공지 패키지와 코어 패키지 조합을 더 명확하게 유지할 수 있게 했습니다.

### 수정

- 공개 목록/상세 페이지에서 번역팩을 거치지 않던 문구를 정리해, 다국어/한글팩 기준 화면 검수가 쉬워지도록 수정했습니다.
- 패치노트에 실제 공개 화면 번역 보강 내용이 빠져 있던 부분을 보완했습니다.

## 4.0.0-beta3 - 2026-07-04

### 안내

- 설치, 공개 라우트, Inertia 브리지 연결 방식을 README에 정리해 패키지를 바로 붙일 수 있도록 문서를 보강했습니다.

### 개선

- 이제 더 이상 `core` 버전선을 직접 대조하지 않아도, `4.0.0-beta3` 기준으로 의존성 메타데이터가 맞춰져 Orbit 패키지 조합을 한결 예측 가능하게 관리할 수 있습니다.
- 공개 목록/상세 페이지와 관리자 엔티티 흐름을 README에 함께 정리해, 설치 후 다음 단계가 문서에 바로 이어지도록 개선했습니다.

### 수정

- 설치 후 어떤 공개 라우트가 열리는지 문서가 비어 있어 직접 코드를 열어봐야 했던 불편을 보완했습니다.
- 패키지 버전 정보가 릴리스 노트 없이 흩어져 보여 운영자가 현재 패키지 선을 판단하기 어려웠던 부분을 정리했습니다.

## 4.0.0-beta2 - 2026-07-04

### 추가

- Orbit 문서 엔진 위에 공지사항 전용 `AnnouncementEntity`와 공개 컨트롤러를 추가했습니다.
- 공지 목록과 상세 페이지를 위한 `announcements.index`, `announcements.show` 공개 라우트를 제공합니다.
- 제목, 슬러그, 본문, 설명, 갤러리, 공개 시각, 승인 상태, 공지/비밀 여부를 관리할 수 있는 관리자 필드를 추가했습니다.

### 개선

- 이제 더 이상 공지 패키지를 붙일 때 문서 섹션과 언어 경로를 수동으로 이어주지 않아도, 패키지 서비스 프로바이더가 Core와 공개 라우트 연결을 함께 처리하도록 개선되었습니다.

### 수정

- 패키지별 번역 경로와 문서 섹션 등록 지점이 분리되어 초기 연결이 불안정하던 부분을 정리했습니다.
